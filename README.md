# yadrovichi

Локальный сервис: часовая запись урока → текст, разбитый по ролям и таймкодам.
Кто говорил, когда и что сказал. Всё на своём железе, без облака.

## Что делаем

1. **Диаризация** — разметить запись по голосам (pyannote, по всему файлу целиком)
2. **Транскрибация** — речь в текст (faster-whisper large-v3-turbo, int8)
3. **Роли** — преподаватель + ученик 1, 2, 3… (граф переходов + маркеры)
4. **Постобработка** — зацикливания, стыки чанков, термины
5. **Выдача** — FastAPI (`/docs`) + Gradio (`/ui`), поток по мере готовности

## Ограничения

Целевое железо: **GTX 1080, 8 GB VRAM, 24 GB RAM**.

| Стадия | Бюджет RTF |
|---|---|
| Транскрибация | ≤ 0.2 (идеал 0.15) |
| Диаризация | ≤ 0.2 |
| Пайплайн | ≤ 0.4 |

⚠️ 1080 — это Pascal. **fp16 там в 64 раза медленнее fp32**, тензорных ядер нет.
`compute_type="float16"` даст замедление, а не ускорение. Только `int8`.


## Прогон на заглушках (GPU не нужен)

```bash
python -m scripts.smoke
```

## Структура

```
src/schema.py       контракт данных — Segment, менять только всей командой
src/stages.py       сигнатуры стадий, реализует МЛщик
src/roles.py        граф переходов + маркеры, GPU не нужен
src/audio.py        приведение к 16 кГц моно, один раз на входе
src/postprocess.py  правила на CPU + normalize_for_wer
src/pipeline.py     конвейер-генератор, чекпоинты, замер RTF
scripts/smoke.py    прогон на фейковых данных
```

## Договорённости

- Кластеризация **одна на весь файл**, не почанково — иначе спикеры
  переименуются на середине
- `min_speakers=2`, верхнюю границу не задавать
- Аудио в гит не коммитить
- Перед WER прогонять **оба** текста через `postprocess.normalize_for_wer`
- DER не заявляем как измеренную метрику: эталона с таймкодами нет
- Колаб только для отладки — там T4, цифры RTF оттуда невалидны
- Застрял на 45 минут — пишешь в чат, не героизируешь

## Порядок работ

1. ✅ Контракт данных и заглушки
2. Диаризация отдельно → сегменты, число спикеров сверить на слух
3. Транскрибация отдельно → текст с таймкодами
4. Склейка + роли → JSON «время, роль, текст»
5. Час аудио проходит целиком ← **это MVP, тегаем и не ломаем**
6. Замеры RTF: три прогона, медиана, по стадиям
7. Постобработка текста
8. FastAPI, модели через lifespan, воркер строго один
9. StreamingResponse
10. Gradio в том же приложении
11. WER на записи с субтитрами, таблица моделей на слайд
