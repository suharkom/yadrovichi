# yadrovichi

Локальный сервис: часовая запись урока → текст, разбитый по ролям и таймкодам.
Кто говорил, когда и что сказал. Всё на своём железе, без облака.

## Что делаем

1. **Диаризация** — разметить запись по голосам (pyannote, по всему файлу целиком)
2. **Транскрибация** — речь в текст (faster-whisper large-v3-turbo, float16)
3. **Роли** — преподаватель + ученик 1, 2, 3… (граф переходов + маркеры)
4. **Постобработка** — зацикливания, стыки чанков, термины, формулы
5. **Выдача** — FastAPI (`/docs`) + Gradio (`/ui`), поток по мере готовности

Стадии 1–3 (`diarize`, `transcribe`, `assign_roles` в `src/stages.py`) — **зона
МЛщика**, сейчас это заглушки с зафиксированными сигнатурами. Рабочая
реализация на GPU (реальные RTF 0.03 / 0.03 / 0.06 на A4000) лежит в истории
git — коммиты `ac0b2c4`, `db0389a`, `088a56c`, оттуда её можно взять как основу.

## Железо

Целевой сервер — **RTX A4000** (Ampere, sm_86, 16 ГБ VRAM, драйвер 595,
CUDA 13.2). Куратор изначально заявлял GTX 1080, но в машине оказалась A4000 —
это меняет выбор точности:

- Ampere имеет тензорные ядра → `compute_type="float16"`. `int8` был бы нужен
  только на Pascal (1080), которого в сервере нет.
- В 16 ГБ whisper и pyannote помещаются одновременно → стадии не выгружаются
  между собой, полный RTF ближе к максимуму стадий, а не к их сумме.

Бюджет RTF задавался под 1080, на A4000 запас шире:

| Стадия | Бюджет | Замер на A4000 |
|---|---|---|
| Транскрибация | ≤ 0.2 | 0.033 |
| Диаризация | ≤ 0.2 | 0.030 |
| Пайплайн | ≤ 0.4 | 0.063 |

## Установка

Нужен **ffmpeg** в PATH — им приводим аудио к 16 кГц моно.

```bash
python -m venv .venv && source .venv/bin/activate
pip install torch torchaudio --index-url https://download.pytorch.org/whl/cu128
pip install -r requirements.txt
hf auth login   # токен Read + принять условия на pyannote/*
```

Проверка железа перед всем остальным:

```bash
python -c "import torch; print(torch.cuda.get_device_capability(0))"   # (8, 6)
```

## Прогон на заглушках (GPU, веса и ffmpeg не нужны)

```bash
python -m scripts.smoke      # диаризация -> чанки -> транскрибация -> роли
python -m scripts.test_api   # приём файла, поток, лимит 200 МБ
```

## Сервер

Тестовый сервер для замеров — доступ в `.env`, в репозиторий не коммитим.
8 ядер CPU, отсюда `POST_WORKERS = 4`.

```bash
uvicorn app.main:app --workers 1   # строго один воркер: видеокарта одна
python -m scripts.bench_rtf data/audio.mp3
python -m scripts.check_transcript data/results/<job_id>.json
```

## Структура

```
src/schema.py        контракт данных — Segment, менять только всей командой
src/stages.py        diarize/transcribe/assign_roles — ЗАГЛУШКИ, пишет МЛщик
src/chunking.py      нарезка по паузам, один спикер на чанк
src/audio.py         приведение к 16 кГц моно, один раз на входе
src/postprocess.py   правила на CPU + normalize_for_wer
src/mathnorm.py      формулы в символьный вид для LLM
src/pipeline.py      конвейер-генератор, очереди, чекпоинты
app/main.py          FastAPI: приём файла, NDJSON-поток, /docs
app/ui.py            Gradio, смонтирован в то же приложение на /ui
scripts/smoke.py     прогон пайплайна на фейковых данных
scripts/test_api.py  проверка API по HTTP
scripts/bench_rtf.py замер RTF по стадиям на реальном файле
scripts/check_transcript.py  проверка: язык, таймлайн, формулы
```

## Договорённости

- Кластеризация **одна на весь файл**, не почанково — иначе спикеры
  переименуются на середине
- `min_speakers=2`, верхнюю границу не задавать
- Аудио в гит не коммитить
- Перед WER прогонять **оба** текста через `postprocess.normalize_for_wer`
- DER не заявляем как измеренную метрику: эталона с таймкодами нет
- Колаб только для отладки — там T4, цифры RTF берём только с сервера (A4000)
- Застрял на 45 минут — пишешь в чат, не героизируешь

## Порядок работ

1. ✅ Контракт данных и заглушки
2. Диаризация отдельно → сегменты, число спикеров сверить на слух *(МЛщик)*
3. Транскрибация отдельно → текст с таймкодами *(МЛщик)*
4. Склейка + роли → JSON «время, роль, текст» *(МЛщик)*
5. Час аудио проходит целиком ← **это MVP, тегаем и не ломаем**
6. Замеры RTF: три прогона, медиана, по стадиям
7. Постобработка текста
8. ✅ FastAPI, воркер строго один
9. ✅ StreamingResponse
10. Gradio в том же приложении
11. WER на записи с субтитрами, таблица моделей на слайд *(МЛщик)*
